# 序列埠監視說明

## 如何查看 ESP32 序列埠輸出

### 方法：使用 PlatformIO 內建監視器

在終端機執行：

```bash
pio device monitor
```

按 `Ctrl+C` 停止監視。

---

## 設定說明

序列埠監視的波特率設定在 `platformio.ini` 中：

```ini
[env:esp32dev]
monitor_speed = 115200
```

這個波特率必須與程式碼中 `Serial.begin()` 的設定一致：

```cpp
Serial.begin(115200);
```

---

## 常見問題

### 看到亂碼？

**開機時的亂碼是正常的**
- ESP32 bootloader 使用 74880 波特率
- 我們的程式使用 115200 波特率
- 所以開機訊息會顯示為亂碼（`ets@ets@...`）
- 等開機完成後就會正常顯示

**持續亂碼**
- 檢查 `platformio.ini` 的 `monitor_speed` 設定
- 檢查程式碼的 `Serial.begin()` 設定
- 確認兩者一致

### 找不到序列埠？

確認：
1. ESP32 已用 USB 連接到電腦
2. 沒有其他程式正在使用序列埠
3. 序列埠驅動程式已安裝

可以用以下指令查看可用的序列埠：

```bash
ls /dev/cu.*
```
