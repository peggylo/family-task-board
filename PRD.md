# 兒童待辦事項互動提醒裝置

## 專案目標
設計一個互動式提醒裝置，幫助 6 歲女兒養成每天完成待辦事項的習慣，透過視覺和聽覺回饋建立成就感。

---

## 核心功能

### 1. 待辦事項清單（8 項）
1. 洗澡
2. 刷牙
3. 收餐袋
4. 收地板玩具
5. **準備明天課程物品**（Google Calendar 聯動）
6. 寫作業（如果有）
7. [待定項目 1]
8. [待定項目 2]

**設計原則**：
- 8 個按鈕固定配置
- 如果當天某項不需要做（如沒作業），仍可按按鈕標記「完成」
- 保持操作一致性，避免複雜判斷

### 2. 互動流程

#### 單項任務操作
```
按下按鈕（第1次）
    ↓
播放 TTS 語音提醒：「記得要洗澡喔！」
    ↓
按下按鈕（第2次）
    ↓
對應的 WS2812 LED 燈亮起（表示完成）
```

#### 全部完成後抽獎流程
```
第 8 個按鈕完成 → 最後一顆燈亮起
    ↓
【恭喜燈光秀】WS2812 全部燈彩虹/閃爍效果（5秒）
    ↓
TTS 語音：「哇！今天全部完成了！你好棒！」
    ↓
【提示抽獎】燈條呼吸燈效果
    ↓
TTS 語音：「現在請你按任意一個按鈕，抽出今日的幸運獎勵！」
    ↓
女兒按下 8 個按鈕中的任意一個
    ↓
【抽獎動畫】（詳見下方）
    ↓
停在某個區域（媽媽/女兒/爸爸）
    ↓
TTS 語音：「這次是【媽媽/女兒/爸爸】的獎勵！」
    ↓
播放該家人預錄的獎勵語音
```

### 接線配置

#### ESP32 → MAX98357A I2S 模組
```
ESP32 GPIO 25  →  BCLK (位元時脈)
ESP32 GPIO 26  →  LRC  (左右聲道時脈)
ESP32 GPIO 22  →  DIN  (音頻數據)
ESP32 5V       →  VIN  (電源)
ESP32 GND      →  GND  (接地)
```

#### MAX98357A → 喇叭
```
MAX98357A SPK+  →  喇叭紅線
MAX98357A SPK-  →  喇叭黑線
```

#### ESP32 → WS2812B LED 燈條
```
ESP32 GPIO 5   →  LED DIN (數據輸入)
ESP32 5V       →  LED 5V
ESP32 GND      →  LED GND
```

#### ESP32 → 按鈕（8 個）
```
按鈕 1-8  →  GPIO 13, 14, 15, 16, 17, 18, 19, 21
各按鈕另一腳接 GND（使用內部上拉電阻）
```

### WS2812 雙用途設計（一排橫向）

**平常模式（任務進度）**：
```
[LED 1] [LED 2] [LED 3] [LED 4] [LED 5] [LED 6] [LED 7] [LED 8]
  洗澡    刷牙    收袋    收玩    準備    作業   項目7   項目8
  
完成後亮綠燈，未完成暗燈或紅燈
```

**抽獎模式（三區域）**：
```
[媽媽區 ][媽媽區 ][媽媽區 ] [女兒區 ][女兒區 ][女兒區 ] [爸爸區 ][爸爸區 ]
 LED 1    LED 2    LED 3     LED 4    LED 5    LED 6     LED 7    LED 8
 紅色     紅色     紅色      藍色     藍色     藍色      黃色     黃色

下方貼對應家人照片
```

**橫向一排的優勢**：
- ✅ 抽獎時三區域視覺連續，效果更明顯
- ✅ 賽馬燈效果從左到右，直觀好看
- ✅ WS2812 燈條本來就是橫向設計
- ✅ 照片可以直接放在對應區域正下方

## 軟體架構

### 語音策略（I2S + 線上串流）

#### 技術架構
使用 ESP32 + MAX98357A I2S 模組直接輸出音頻，支援：
- Google Cloud Text-to-Speech API（即時生成語音）
- Google Drive 預錄檔案串流播放
- 無需 SD 卡，所有音頻透過網路處理

#### 三種語音來源

**1. TTS 即時生成（系統提示和任務提醒）**
```cpp
audio.connecttospeech("記得要洗澡喔", "zh-TW");
audio.connecttospeech("哇！今天全部完成了！你好棒！", "zh-TW");
audio.connecttospeech("現在請按任意按鈕，抽出今日的幸運獎勵", "zh-TW");
```

**應用場景**：
- 8 項任務的第一次按鈕提醒
- 全部完成的恭喜語音
- 抽獎提示語音
- Google Calendar 課程提醒（內容動態生成）

**延遲**：0.8-1.5 秒（可接受，因為是主動提醒）

**2. Google Drive 預錄檔案（家人獎勵）**
```cpp
// 從 Google Drive 公開連結串流播放
audio.connecttohost("https://drive.google.com/uc?export=download&id=xxxxx");
```

**應用場景**：
- 媽媽錄製的獎勵語音（5-10 個）
- 女兒自己錄製的獎勵語音（5-10 個）
- 爸爸錄製的獎勵語音（5-10 個）

**延遲處理**：
- 按下抽獎按鈕時立即開始串流下載
- 同時播放 5 秒燈光動畫
- 動畫結束時音檔已緩衝完成，立即播放
- **實際體驗：幾乎無延遲**

**3. 備用方案（可選）**
如果網路不穩定，可以預先下載常用獎勵到 ESP32 的 SPIFFS：
```cpp
// 開機時預載
if (WiFi.status() == WL_CONNECTED) {
  preloadRewardsToSPIFFS();
}

// 使用時優先播放本地快取
if (fileExistsInSPIFFS("mom_reward_1.mp3")) {
  audio.connecttoFS(SPIFFS, "mom_reward_1.mp3");
} else {
  audio.connecttohost(driveURL);  // 降級到線上串流
}
```

#### 延遲時間軸（抽獎情境）

```
T=0s    按下抽獎按鈕
        ├─ 程式決定結果（媽媽/女兒/爸爸）
        ├─ 開始串流下載對應音檔
        └─ 啟動燈光動畫
        
T=0-1s  階段1：全部燈快速閃爍
        └─ 背景持續下載（已下載 ~50KB）
        
T=1-3s  階段2：三區域快速輪流
        └─ 背景持續下載（已下載 ~120KB）
        
T=3-5s  階段3：逐漸變慢
        └─ 背景持續下載（已完成 160KB）
        
T=5s    動畫結束
        ├─ 檢查：音檔已緩衝完成 ✓
        ├─ TTS 宣布結果：「這次是媽媽的獎勵！」
        └─ 立即播放預錄獎勵
```

**檔案大小估算**：
- 5-10 秒語音，128kbps MP3
- 檔案大小：80-160 KB
- WiFi 下載時間（2 Mbps）：0.6-1.2 秒
- **5 秒動畫綰綰有餘**

#### Google Drive 檔案組織

```
Google Drive 資料夾結構：
/兒童提醒裝置/

   ```

   ```
   轉換：https://drive.google.com/uc?export=download&id=FILE_ID
   ```
3. 將連結寫入程式或存在雲端設定檔

#### 程式庫需求

```cpp
// ESP32 音頻處理

// WiFi 和 HTTP
#include <WiFi.h>
#include <HTTPClient.h>

void setup() {
  // I2S 接腳配置

  // WiFi 連線

  while (WiFi.status() != WL_CONNECTED) {

  }

```

### Google Calendar 聯動

**功能**：「準備明天課程物品」按鈕（第 5 項）觸發時
1. ESP32 透過 WiFi 查詢 Google Calendar（或讀取預先準備的資料）
2. 取得明天的課程資訊
3. 根據課程名稱產生對應提醒文字
4. 透過 TTS 播放語音

**實作方式**（建議）：
```

    ↓

    ↓

    ↓
```

**範例**：

## 抽獎動畫設計

### 動畫流程（總時長約 6-7 秒）

#### 階段 1：全部燈快速閃爍（1 秒）
```
所有 8 顆燈同時快速閃爍
速度：每 100ms 切換開關
效果：營造緊張刺激的氛圍
```

#### 階段 2：三區域輪流亮（2 秒，快速）
```
[媽媽區亮] [女兒區暗] [爸爸區暗]  → 150ms
[媽媽區暗] [女兒區亮] [爸爸區暗]  → 150ms
[媽媽區暗] [女兒區暗] [爸爸區亮]  → 150ms
重複輪流...
```

#### 階段 3：逐漸變慢（2-3 秒）
```
輪流速度從 150ms 逐漸增加到 500ms
視覺上越來越慢，增加期待感
```

#### 階段 4：最終結果（1 秒）
```
停在某個區域（程式預先決定）
該區域的燈持續亮著
閃爍 3 次確認
播放「叮！」音效
```

### 顏色配置
- **媽媽區**（LED 1-3）：紅色
- **女兒區**（LED 4-6）：藍色  
- **爸爸區**（LED 7-8）：黃色

### 程式邏輯
```cpp
// 偽代碼
void lotteryAnimation() {
  // 1. 先決定結果
  int winner = random(0, 3);  // 0=媽媽, 1=女兒, 2=爸爸
  
  // 2. 全部閃爍
  fastBlinkAll(1000);  // 1 秒
  
  // 3-4. 三區域輪流，逐漸變慢
  int speed = 150;  // 初始 150ms
  for (int round = 0; round < 20; round++) {
    int zone = round % 3;  // 0, 1, 2 輪流
    showZone(zone);
    delay(speed);
    speed += 15;  // 每次增加 15ms，越來越慢
  }
  
  // 5. 顯示最終結果
  showZone(winner);
  delay(500);
  
  // 6. 閃爍確認 3 次
  for (int i = 0; i < 3; i++) {
    allOff();
    delay(200);
    showZone(winner);
    delay(200);
  }
  
  // 7. 保持該區亮著
  showZone(winner);
  
  // 8. 播放結果
  playTTS("這次是" + getName(winner) + "的獎勵");
  delay(1500);
  playReward(winner);  // 播放預錄獎勵
}
```

#### 製作步驟

**材料準備**：
- 厚紙板（5mm）或壓克力板（3-5mm）- A4 大小
- 描圖紙或烘焙紙（半透明，用於擴散燈光）
- 美工刀、剪刀、尺
- 熱熔膠或雙面膠
- 裝飾材料（貼紙、彩色筆等）

**步驟 1：設計版面**
- 用鉛筆在紙板上畫出 8 個方格（一排橫向）
- 每格尺寸：2.5 x 2.5 cm
- 格子間距：0.5 cm
- 總寬度：約 24 cm（適合 A4 紙寬度）

**步驟 2：挖洞**
- 用美工刀沿線切出方形洞
- 邊緣打磨平滑
- 可讓女兒協助裝飾邊框

**步驟 3：貼半透明紙**
- 在洞口內側貼上描圖紙
- 這會讓燈光柔和擴散，避免刺眼
- 固定方式：用透明膠帶或膠水

**步驟 4：固定燈條**
- WS2812 燈條貼在紙板背面
- 確保每顆 LED 對準一個洞（可先測試點亮確認位置）
- 用熱熔膠或雙面膠固定
- 注意：燈條與紙板距離約 0.5-1 cm，燈光效果更好

**步驟 5：裝飾**
- 面板外側讓女兒自由裝飾
- 三個區域下方貼上家人照片（5x5 cm）
- 每個格子旁標示任務名稱
- 可加上星星、愛心等裝飾元素

```
女兒回家後
    ↓

    ↓
按對應按鈕（第1次）→ 聽到 TTS 提醒 → 去完成
```

#### 現場檢查項目

**MAX98357A 模組**：
- ✅ 確認有 5 個針腳（BCLK, LRC, DIN, VIN, GND）
- ✅ 確認有喇叭接線座（+ / - 或 SPK+/SPK-）
- ✅ 詢問是否測試過（避免買到瑕疵品）

**WS2812B 燈條**：
- ✅ 確認是「可編址」LED（每顆可獨立控制顏色）
- ✅ 有 3 條線：5V、GND、DIN（資料線）
- ✅ 如買 1 米長的，可請老闆幫忙剪成 8 顆（或買整條回家自己剪）
- ✅ 確認工作電壓 5V

**輕觸按鈕**：
- ✅ 測試按壓手感
- ✅ 多買 2 個備用（很便宜）
- ✅ 4 支腳的（標準規格）

**杜邦線**：
- ✅ 長度 20cm 左右即可
- ✅ 公對公、公對母各買一包（每包約 40 條）

### 音頻檔案準備

**家人錄音流程**：
1. 用手機錄音 App 錄製（建議：Voice Memos / 錄音機）
2. 錄製格式：盡量清晰、安靜環境
3. 上傳到 Google Drive 共享資料夾
4. 設定為「知道連結的任何人都可查看」
5. 取得分享連結並轉換格式：
   ```
   原始連結：
   https://drive.google.com/file/d/FILE_ID/view?usp=sharing
   
   轉換後（程式使用）：
   https://drive.google.com/uc?export=download&id=FILE_ID
   ```

**音檔規格建議**：
- 格式：MP3（128 kbps）
- 長度：5-10 秒
- 檔案大小：80-160 KB
- 每位家人：5-10 個不同獎勵

**錄音內容範例**：
- 媽媽：「今天好棒！晚上可以多講一個故事」
- 女兒：「我全部做完了！我好厲害！」
- 爸爸：「太棒了！可以多玩 15 分鐘」
